  • setBankAccountType()
  • setChargeAmount() — A positive decimal value less than 1,000,000 to charge against the credit card or bank account. Default currency can be overwritten with `currency` field. Leave blank or set to 0 to tokenize.
  • setCurrency() — The currency of the `charge_amount`, formatted as [ISO 4127](http://www.iso.org/iso/home/standards/currency_codes.htm) format. The default value is `USD` and all values allowed from the Pay Connector are accepted.
  • setDescription() — The description can be used to provide more details regarding the transaction. This information is submitted along with the payment details to the Payment Connector which are then posted on the transactions.
  • setInput() — A list of inputs that should be accepted. Currently only `dtmf` is supported. All digits captured during a pay session are redacted from the logs.
  • setMinPostalCodeLength() — A positive integer that is used to validate the length of the `PostalCode` inputted by the user. User must enter this many digits.
  • setParameter() — A single-level JSON object used to pass custom parameters to payment processors. (Required for ACH payments). The information that has to be included here depends on the <Pay> Connector. [Read more](https://www.twilio.com/console/voice/pay-connectors).
  • setPaymentConnector() — This is the unique name corresponding to the Pay Connector installed in the Twilio Add-ons. Learn more about [<Pay> Connectors](https://www.twilio.com/console/voice/pay-connectors). The default value is `Default`.
  • setPaymentMethod()
  • setPostalCode() — Indicates whether the credit card postal code (zip code) is a required piece of payment information that must be provided by the caller. The default is `true`.
  • setSecurityCode() — Indicates whether the credit card security code is a required piece of payment information that must be provided by the caller. The default is `true`.
  • setTimeout() — The number of seconds that <Pay> should wait for the caller to press a digit between each subsequent digit, after the first one, before moving on to validate the digits captured. The default is `5`, maximum is `600`.
  • setTokenType()
  • setValidCardTypes() — Credit card types separated by space that Pay should accept. The default value is `visa mastercard amex`
